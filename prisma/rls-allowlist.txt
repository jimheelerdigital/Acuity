# appear here with an explicit "rls" or "no-rls" annotation.
#
# Format:
#   ModelName rls          # RLS enabled in prod (default; safe)
#   ModelName no-rls       # intentionally exposed; short reason after #
#
# Adding a new model? Add a line here AND, if marked "rls", run:
#   ALTER TABLE public."ModelName" ENABLE ROW LEVEL SECURITY;
# in production after the schema push.
#
# Background: 2026-04-29 incident — six tables shipped without RLS,
# three contained PII, exposed via the public anon key. See
# docs/launch-audit-2026-04-26/14-rls-prevention.md.

# This file is enforced by scripts/check-rls-coverage.ts and the
# .github/workflows/rls-check.yml CI workflow.
