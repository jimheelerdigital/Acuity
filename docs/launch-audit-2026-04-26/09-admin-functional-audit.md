# 09 — Admin functional audit (tab-by-tab)

**Status:** functional walkthrough of `/admin` against `prisma/schema.prisma` and the live API routes.
**Date:** 2026-04-27
**Goal:** know which tabs render real production data, which are fully wired-but-empty (will populate as data accrues), and which are missing pieces a launching SaaS would want.

## Reading guide

- **Real data** — query hits prod tables that are actively written today. Numbers will be live the moment users do the corresponding action.
- **Wired but empty** — query hits a real prod table, but that table has no upstream writer or the writer just hasn't run yet. UI works, data is `0`/`[]` until the upstream lands.
- **Hardcoded** — value is a literal in the code, not from the DB.
- **Mocked / stub** — no data path at all; placeholder text or `—`.

`getCached()` wraps every `/api/admin/metrics` query with a per-tab TTL (Overview 5m, Revenue 10m, Funnel 15m, Ads 15m, AI Costs 2m, Content Factory live, Users 2m, Feature Flags 1m, Guide ∞). `Refresh` button on each tab calls the same endpoint with `?refresh=true` to bust cache.

## Cross-cutting gaps a launching SaaS usually wants

Surfaced here so they don't get re-listed under each tab. None are blockers; all are sub-week additions.

1. **Signup-source attribution.** Without `User.signupSource`, the Funnel "drop-off by source" view, Growth source-mix, and per-campaign Ads CAC are all blocked. Adding a single string column and writing it from waitlist/UTM unlocks four downstream metrics at once.
2. **Real MRR from Stripe invoices, not `count × $12.99`.** Flat-price math undercounts annual plans and overcounts at any discount. A nightly Inngest job that pulls `invoice.paid` aggregates and writes a `RevenueSnapshot` row would also give you a historical MRR chart for free.
3. **Cohort retention (D1, D7, D30).** Currently only D0 is computed. D7 is the single most predictive engagement metric for this kind of habit product (per the Guide itself). Same query shape as the Funnel "Active Day 7" but emitted as a per-week-cohort series.
4. **Status-string drift.** Two tabs (Revenue, Users) check `subscriptionStatus === "PRO"`. Search the codebase for the actual enum values written by the Stripe webhook before extending — looks like `ACTIVE` is what's being persisted, so the green pill never shows up for paying users.
5. **Hardcoded thresholds.** AI budget ($100/mo), monthly price ($12.99 in API and $9.99 in Overview UI), and admin email (`adminUserId` only — no central admin contact env var) are scattered. A single `app/admin/config.ts` (or env) would let Keenan tune without a code change.
6. **Period-over-period comparisons are broken in two places.** Overview's `prevPayingSubs` query is a copy of `payingSubs`, and Revenue has no period series at all. To make the metric cards' green/red deltas meaningful, you need a `MetricSnapshot` table that captures `payingSubs / mrr / dau / wau / mau` once a day.
7. **No "Notifications" tab.** Push-notification delivery rates, opt-in %, daily-reminder fire success — all dark right now. Mobile push is the listed retention lever in the Guide; no dashboard for it.
8. **No "Errors / Health" tab.** Sentry has its own UI, but a 1-tile "errors per 1k requests today" hooked to Sentry's API would catch regressions before users do. Closest equivalent is the Red Flags scanner.

## What's actually production-grade as-is

To balance the gap list — these tabs are good-to-go and shouldn't be touched on a "fix the admin" pass:

- **Content Factory** — full CRUD, real-time job status, multi-step pipeline.
- **Feature Flags** — flag CRUD UI is missing, but the override flow, audit logging, and tier gating are clean.
- **AI Costs** — query is right, table is granular, line chart is honest. Just add the budget knob and `% of MRR` derived metric.
- **Trial Emails** — once the Resend webhook lands the open/click cells light up; the rest is right.
- **Users** detail modal — Stripe + Sentry deep-links, audit-logged actions, override visibility. Solid CS console.
- **Guide** — beats writing this in Notion.

## Notes

- All `/api/admin/*` reads honor the `requireAdmin()` (or inline equivalent) gate. Nothing on this list is exposed unauthenticated. The audit didn't re-verify each route — see `08-admin-audit.md` for that.
- Several `_meta` fields (`cached`, `computedAt`, `durationMs`) are returned by the metrics endpoint but only the Refresh button surfaces them. Could be useful in a "system health" header.
- "Wired but empty" status means the UI will start showing real data the moment the upstream emitter runs (e.g., Red Flags appear when `scan-red-flags.ts` fires; Trial Email opens appear when the Resend webhook is configured). It doesn't mean the code is broken.
